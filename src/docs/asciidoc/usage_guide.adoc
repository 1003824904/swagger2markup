== Usage guide

Swagger2Markup converts a Swagger JSON or YAML specification into either **AsciiDoc**, **GitHub Flavored Markdown** or **Atlassian Confluence Wiki** documents. By default the Swagger2Markup converts a Swagger specification into four AsciiDoc files: __overview.adoc__, __paths.adoc__ , __security.adoc__ and __definitions.adoc__. But you can also convert a Swagger specification into only one file or a String.

=== Available Swagger2Markup properties

The following tables list all available Swagger2Markup properties:

[options="header"]
|===
|Name | Description |  Possible Values | Default
|swagger2markup.markupLanguage| Asd | ASCIIDOC, MARKDOWN, CONFLUENCE_MARKUP | ASCIIDOC
|swagger2markup.swaggerMarkupLanguage| Asd | ASCIIDOC, MARKDOWN, CONFLUENCE_MARKUP | MARKDOWN
|swagger2markup.pathsGroupedBy| Group the path operations | AS_IS, TAGS | AS_IS
|swagger2markup.outputLanguage| The output language | EN, DE, FR, RU | EN
|swagger2markup.lineSeparator| The line separator which should be used | UNIX, WINDOWS, MAC | System default
|swagger2markup.generatedExamplesEnabled| Generate HTTP request and response examples | true, false | false
|swagger2markup.inlineSchemaDepthLevel| The depth for inline schemas | Any Integer | 0
|swagger2markup.flatBodyEnabled| Render a flat request body | true, false | false
|===

==== Configure the order of Swagger objects

[options="header"]
|===
|Name | Description |  Possible Values | Default
|swagger2markup.tagOrderBy| The order of global tags | AS_IS, NATURAL, CUSTOM | NATURAL
|swagger2markup.operationOrderBy| The order of path operations | AS_IS, NATURAL, CUSTOM | NATURAL
|swagger2markup.definitionOrderBy| The order of definitons | AS_IS, NATURAL, CUSTOM | NATURAL
|swagger2markup.parameterOrderBy| The order of operation parameters | AS_IS, NATURAL, CUSTOM | NATURAL
|swagger2markup.propertyOrderBy | The order of definition properties | AS_IS, NATURAL, CUSTOM | NATURAL
|swagger2markup.responseOrderBy| The order of responses | AS_IS, NATURAL, CUSTOM | NATURAL
|===

==== Configure the document file names

[options="header"]
|===
|Name | Description |  Possible Values | Default
|swagger2markup.overviewDocument| The file name of the overview document | Any String | overview
|swagger2markup.pathsDocument| The file name of the paths document | Any String | paths
|swagger2markup.definitionsDocument| The file name of the definitions document | Any String | definitions
|swagger2markup.securityDocument| The file name of the security document | Any String | security
|===

=== Enable the generation of separated files

[options="header"]
|===
|Name | Description |  Possible Values | Default
|swagger2markup.separatedDefinitionsEnabled| Create a file for each definition | true, false | false
|swagger2markup.separatedOperationsEnabled| Create a file for each path operation | true, false | false
|swagger2markup.separatedOperationsFolder| The target folder path for definition files| Any valid path | operations
|swagger2markup.separatedDefinitionsFolder| The target folder path operation files | Any valid path | definitions
|===


==== Configure interdocument cross references

[options="header"]
|===
|Name | Description |  Possible Values | Default

|swagger2markup.interDocumentCrossReferencesPrefix| A custom prefix for cross references | Any String | 
|swagger2markup.anchorPrefix| Configure a custom anchor prefix | Any String | 
|===
